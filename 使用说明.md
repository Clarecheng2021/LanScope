# LanScope 资产扫描工具使用说明

LanScope 是一个轻量级内网资产扫描工具，用于发现主机、扫描 TCP 端口，并采集 HTTP/HTTPS 服务的基础信息。

请只扫描你拥有或已获得授权的网络和设备。

## 1. 运行环境

需要 Python 3.10 或更高版本。

如果你的系统已经安装 Python，可以在项目目录执行：

```powershell
python -m lanscope --help
```

当前这台机器的系统 PATH 里可能没有 `python` 命令。如果你在 Codex 环境里测试，可以使用内置 Python：

```powershell
C:\Users\Administrator\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe -m lanscope --help
```

以下示例默认使用 `python`。如果你的机器识别不了 `python`，把命令里的 `python` 换成上面的完整路径即可。

## 2. 查看帮助

```powershell
python -m lanscope --help
```

你应该能看到参数说明，例如：

- `targets`：扫描目标，可以是 IP、域名、CIDR 网段
- `--ports`：端口范围
- `--timeout`：连接超时时间
- `--concurrency`：并发数量
- `--skip-ping`：跳过 ping，直接扫描端口
- `--format`：输出格式，支持 `table`、`json`、`csv`
- `--output`：输出到文件
- `--interactive`：交互式输入扫描参数

## 3. 快速测试

### 3.1 扫描本机

```powershell
python -m lanscope 127.0.0.1 --ports 80,443 --skip-ping
```

如果本机没有开放 80 或 443 端口，结果里 `services` 为空是正常的。

### 3.2 扫描一个小网段

建议先用很小的测试网段，例如 `/30` 或 `/29`：

```powershell
python -m lanscope 192.168.1.0/30 --ports 22,80,443 --timeout 1 --skip-ping
```

说明：

- `/30` 通常只会扫描 2 个可用主机地址
- `--skip-ping` 可以避免目标禁 ping 时被误判为不可达
- `--timeout 1` 可以让测试更快结束

### 3.3 扫描常见端口

```powershell
python -m lanscope 192.168.1.10 --ports common --skip-ping
```

`common` 会扫描常见服务端口，例如 SSH、HTTP、HTTPS、MySQL、RDP、Redis、MongoDB 等。

### 3.4 扫描 Top 100 端口

```powershell
python -m lanscope 192.168.1.10 --ports top100 --skip-ping
```

这个模式比 `common` 更全面，但扫描时间和连接数量也会增加。

### 3.5 自定义端口

```powershell
python -m lanscope 192.168.1.10 --ports 22,80,443,8000-8010 --skip-ping
```

端口格式支持：

- 单个端口：`80`
- 多个端口：`22,80,443`
- 端口范围：`8000-8010`
- 混合写法：`22,80,443,8000-8010`

### 3.6 按 nmap 思路进行全端口扫描

如果需要盲扫 TCP 全端口，不建议直接对整个网段的每台主机都扫 `1-65535`。新版提供了扫描模式：

```powershell
python -m lanscope 192.168.1.0/24 --mode full
```

`full` 模式会先做主机发现，再只对发现的存活主机扫描 `1-65535` 端口。这样比“全网段 × 全端口”暴力扫描更快，也更不容易压垮网络。

可用模式：

- `quick`：主机发现 + 少量高价值端口
- `normal`：主机发现 + top100 端口
- `full`：主机发现 + TCP 全端口 `1-65535`
- `deep`：更保守的全端口深度扫描
- `custom`：使用你自己指定的 `--ports`

全端口扫描建议参数：

```powershell
python -m lanscope 192.168.1.0/24 --mode full --timeout 0.4 --global-concurrency 800 --per-host-concurrency 100
```

如果你明确要跳过主机发现，对指定主机直接扫端口：

```powershell
python -m lanscope 192.168.1.10 --ports full --no-discover-hosts --timeout 0.4
```

## 4. 输出格式

### 4.1 表格输出

默认输出格式是表格：

```powershell
python -m lanscope 127.0.0.1 --ports 80,443 --skip-ping
```

表格字段含义：

- `HOST`：输入的目标
- `ADDRESS`：解析后的 IP 地址
- `UP`：是否可达
- `PORT`：开放端口
- `SERVICE`：推测的服务名称
- `HTTP`：HTTP 状态码
- `TITLE/BANNER`：网页标题或服务 banner

### 4.2 JSON 输出

```powershell
python -m lanscope 127.0.0.1 --ports 80,443 --skip-ping --format json
```

保存到文件：

```powershell
python -m lanscope 127.0.0.1 --ports 80,443 --skip-ping --format json --output scan.json
```

### 4.3 CSV 输出

```powershell
python -m lanscope 127.0.0.1 --ports 80,443 --skip-ping --format csv --output scan.csv
```

CSV 文件可以用 Excel、WPS 或其他表格工具打开。

## 5. 交互式模式

如果不想记命令参数，可以使用交互模式：

```powershell
python -m lanscope --interactive
```

它会依次询问：

- 扫描目标
- 端口范围
- 超时时间
- 是否跳过 ping
- 输出格式
- 输出文件路径

## 6. Web 控制台

如果你不想手动查看本机 IP，也不想自己换算 CIDR 网段，可以启动 Web 控制台：

```powershell
python -m lanscope.web
```

启动后在浏览器打开：

```text
http://127.0.0.1:8765
```

Web 控制台提供：

- 自动发现本机局域网地址
- 自动推荐当前局域网网段，例如 `192.168.1.0/24`
- 一键扫描当前局域网
- 自定义目标和端口扫描
- 查看扫描任务进度
- 查看 JSON 扫描结果

快速扫描默认端口为：

```text
22,80,443,445,3389,8080
```

这些端口覆盖 SSH、HTTP、HTTPS、SMB、远程桌面和常见 Web 管理端口，适合快速发现局域网内的设备。

## 7. 推荐测试流程

第一次测试建议按这个顺序：

1. 查看帮助：

```powershell
python -m lanscope --help
```

2. 扫描本机：

```powershell
python -m lanscope 127.0.0.1 --ports 80,443 --skip-ping
```

3. 扫描一个确定存在的内网 IP：

```powershell
python -m lanscope 你的内网IP --ports 22,80,443,3389 --skip-ping
```

4. 导出 JSON：

```powershell
python -m lanscope 你的内网IP --ports common --skip-ping --format json --output scan.json
```

5. 导出 CSV：

```powershell
python -m lanscope 你的内网IP --ports common --skip-ping --format csv --output scan.csv
```

## 8. 参数建议

### 小范围测试

```powershell
python -m lanscope 192.168.1.0/30 --ports 22,80,443 --timeout 1 --concurrency 16 --skip-ping
```

### 单台主机详细测试

```powershell
python -m lanscope 192.168.1.10 --ports top100 --timeout 1.5 --skip-ping --format json
```

### 多台主机导出资产清单

```powershell
python -m lanscope 192.168.1.1 192.168.1.10 192.168.1.20 --ports common --skip-ping --format csv --output assets.csv
```

## 9. 常见问题

### 提示无法识别 `python`

说明 Python 没有安装，或者没有加入 PATH。

你可以：

- 安装 Python 3.10+
- 或在 Codex 环境中使用内置 Python 完整路径

Codex 内置 Python 示例：

```powershell
C:\Users\Administrator\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe -m lanscope 127.0.0.1 --ports 80,443 --skip-ping
```

### 扫描结果没有开放端口

可能原因：

- 目标确实没有开放这些端口
- 本机或目标防火墙拦截了连接
- 端口不在你指定的扫描范围内
- 超时时间太短

可以尝试：

```powershell
python -m lanscope 目标IP --ports top100 --timeout 2 --skip-ping
```

### 为什么建议加 `--skip-ping`

很多主机禁用了 ICMP ping，但 TCP 端口仍然开放。测试阶段加上 `--skip-ping`，可以减少误判。

### 扫描大网段要注意什么

不要一开始就扫 `/16`、`/8` 这类大范围网段。建议先扫小网段，例如：

```powershell
python -m lanscope 192.168.1.0/30 --ports 22,80,443 --skip-ping
```

确认行为符合预期后，再逐步扩大范围。

## 10. 当前版本注意事项

当前版本适合做基础资产发现，不包含漏洞利用、弱口令爆破或高风险检测。

已知需要后续增强的地方：

- 更严格的全局并发控制
- 更安全的大网段预检查
- 更完善的 HTTP 指纹识别
- 更丰富的服务协议识别
- Web 管理界面和历史扫描结果存储
